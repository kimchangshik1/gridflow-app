- D_OPERATIONS_RUNBOOK.txt
  - 실제 서버 운영 구성, 백업, 복구, 로그, alert 점검 기준이다.

  - 납품 범위와 비포함 범위를 다시 확인할 때 본다.

장애 시 먼저 볼 문서

  - 서비스 상태, 로그, API 키, 거래소 응답, 백업 상태의 1차 점검 순서가 있다.

문서 사용 메모
- 고객 전달 우선 문서는 scope, handoff runbook, operations runbook 순서로 보는 것이 적합하다.
- 설치 재현 또는 출시 판정은 install checklist와 release quality gates를 함께 봐야 한다.
- 공식 현재 PASS/HOLD/FAIL 결론 문서는 저장소에서 별도로 확인하지 못했다. 최신 판정은 GRIDFLOW_RELEASE_EVIDENCE.txt에 미확인으로 기록한다.
