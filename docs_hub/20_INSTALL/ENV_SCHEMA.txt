5. monitor_config.json Schema

Location:
- default path: /etc/gridflow/monitor_config.json
- override path env: MONITOR_CONFIG_PATH

Observed keys now:
- exchange
- api_key
- api_secret
- tier

Schema:
- exchange | json | orderlens-ops | app/monitor/auth.py has_saved_monitor_config(); app/monitor/repo.py _make_client() | required for configured monitor state | no | empty makes monitor config treated as unsaved; /monitor redirects to /monitor/setup
- api_key | json | orderlens-ops | app/monitor/auth.py has_saved_monitor_config(); app/monitor/repo.py _make_client() | required for configured monitor state | yes | empty makes monitor config treated as unsaved; exchange client cannot be constructed with configured key
- api_secret | json | orderlens-ops | app/monitor/auth.py has_saved_monitor_config(); app/monitor/repo.py _make_client() | required for configured monitor state | yes | empty makes monitor config treated as unsaved; exchange client cannot be constructed with configured secret
- tier | json | orderlens-ops | app/monitor/tier.py get_current_tier() | optional | no | missing/invalid value falls back to lite

Schema notes:
- app/monitor/auth.py _MONITOR_CONFIG_FIELDS only guarantees exchange/api_key/api_secret when creating an empty config payload.
- tier is a real read key but not part of _MONITOR_CONFIG_FIELDS; it may be absent from newly cleared config.
- Allowed tier values proven by app/monitor/tier.py: lite, pro, signature.
