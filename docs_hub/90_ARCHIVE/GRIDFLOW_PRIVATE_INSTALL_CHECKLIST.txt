GridFlow 설치 완료 체크리스트

1. 문서 목적

이 문서는 GridFlow 설치자가 납품 전 설치 완료 상태를 확인하기 위한 점검표다.
고객에게 전달하기 전, 설치 누락과 기본 운영 오류를 줄이는 데 사용한다.

[ ] 설치 완료 확인용 문서로 사용했다.
[ ] 납품 전 최종 점검표로 사용했다.
[ ] 미완료 항목은 아래 설치 완료 판정란에 기록했다.


2. 서버/접속 기본

[ ] 서버 SSH 접속이 가능하다.
[ ] 서버 재부팅 후에도 접속이 가능하다.
[ ] 도메인이 서버로 정상 연결된다.
[ ] HTTPS 접속이 가능하다.
[ ] 브라우저 주소창에 인증서 경고가 없다.
[ ] 80 -> 443 리다이렉트가 의도대로 동작한다.
[ ] 필요한 포트만 열려 있다.
[ ] 방화벽 또는 보안 그룹 설정을 확인했다.

메모:


3. 서비스 기동 확인

필요한 서비스만 active(running) 상태인지 확인한다.
중복 서비스나 사용하지 않는 서비스가 켜져 있으면 납품 전 정리한다.

[ ] gridflow-app.service 상태가 active(running)이다.
[ ] upbit-bot.service 상태가 active(running)이다.
[ ] orderlens-ops.service 상태가 active(running)이다.
[ ] nginx.service 상태가 active(running)이다.
[ ] postgresql@16-main.service 상태가 active 상태다.
[ ] upbit-web.service는 현재 운영 경로가 아니며, 존재하더라도 disabled/inactive 상태다.
[ ] 사용하지 않는 중복 웹 서비스가 떠 있지 않다.
[ ] 재시작 루프가 없다.
[ ] permission denied 오류가 없다.
[ ] env 또는 secret 파일 로딩 오류가 없다.

메모:


4. 로그인/접속 확인

[ ] 메인 화면 접속이 가능하다.
[ ] 로컬 앱 endpoint http://127.0.0.1:8000/ 가 HTTP 200으로 응답한다.
[ ] 로컬 모니터 endpoint http://127.0.0.1:8010/monitor 가 로그인 전 /monitor/login으로 이동한다.
[ ] HTTPS 메인 endpoint https://gridflow.co.kr/ 가 HTTP 200으로 응답한다.
[ ] HTTPS 모니터 endpoint https://gridflow.co.kr/monitor 가 로그인 전 /monitor/login으로 이동한다.
[ ] 로그인 화면 접속이 가능하다.
[ ] 고객 계정으로 로그인할 수 있다.
[ ] 관리자 또는 운영 계정 권한을 확인했다.
[ ] 권한 없는 경로 접근이 차단된다.
[ ] 로그아웃 후 보호 화면에 바로 접근할 수 없다.

메모:


5. 거래소/API 키 확인

[ ] 거래소 API 키 저장 경로를 확인했다.
[ ] 고객별 키가 고객별 계정에 연결되어 있다.
[ ] 공용 env 키에 의존해 주문하지 않는다.
[ ] 키 변경 후 서비스에 반영되는지 확인했다.
[ ] 잘못된 키 사용 시 오류가 명확히 표시된다.
[ ] API 키 원문을 문서나 로그에 남기지 않았다.
[ ] 거래 권한과 IP 제한 설정을 확인했다.

메모:


6. 모드 확인

[ ] GUEST 상태와 고객 운영 계정을 구분할 수 있다.
[ ] DRY RUN 상태가 화면 또는 설정에서 구분된다.
[ ] LIVE 상태가 화면 또는 설정에서 구분된다.
[ ] 테스트 계정이 LIVE 주문에 사용되지 않는다.
[ ] 고객이 의도한 모드로 설정되어 있다.
[ ] 모드 변경 후 주문 경로를 다시 확인했다.

메모:


7. 주문/전략 기본 확인

[ ] 주문 화면에 진입할 수 있다.
[ ] 최근 주문 목록을 확인할 수 있다.
[ ] 전략 목록을 조회할 수 있다.
[ ] 전략 생성 또는 기본 설정 저장이 가능하다.
[ ] DCA 전략 조회가 가능하다.
[ ] Grid 전략 조회가 가능하다.
[ ] Rebalancing 전략 조회가 가능하다.
[ ] read-only 또는 monitor 화면에 접속할 수 있다.
[ ] 주문 상태가 화면과 DB 흐름에서 크게 어긋나지 않는다.

메모:


8. 로그/모니터링 확인

[ ] 최근 주문 로그를 확인할 수 있다.
[ ] 최근 활동 로그를 확인할 수 있다.
[ ] systemd journal 로그를 확인할 수 있다.
[ ] 앱 오류 로그를 확인할 수 있다.
[ ] 봇 오류 로그를 확인할 수 있다.
[ ] invalid_access_key 반복 오류가 없다.
[ ] RemainingReqParsingError 반복 출력이 없다.
[ ] DB 연결 오류가 없다.
[ ] rate limit 또는 거래소 응답 오류가 반복되는지 확인했다.

메모:


9. 백업/복구/로테이션 확인

[ ] DB 백업 파일 생성 경로를 확인했다.
[ ] 최근 백업 파일이 존재한다.
[ ] 백업 파일 크기가 비정상적으로 작지 않다.
[ ] 백업 자동 실행 방식이 등록되어 있다.
[ ] 백업 timer 또는 cron이 존재한다.
[ ] 복구 검증 절차를 1회 확인했다.
[ ] 복구 검증이 운영 DB를 덮어쓰지 않는 방식임을 확인했다.
[ ] logrotate 설정이 존재한다.
[ ] 로그가 과도하게 누적되지 않도록 설정되어 있다.
[ ] health alert 서비스 또는 타이머가 존재한다.
[ ] health alert가 어떤 방식으로 장애를 알리는지 확인했다.

메모:


10. 납품 제외 항목 재확인

고객에게 아래 항목이 기본 납품 범위가 아님을 설명한다.

[ ] 운영 대행 상품이 아님을 설명했다.
[ ] 수익 보장 상품이 아님을 설명했다.
[ ] 24시간 모니터링 상품이 아님을 설명했다.
[ ] 실시간 매매 판단 대행이 아님을 설명했다.
[ ] 거래소 정책 변경 상시 대응이 기본 포함이 아님을 설명했다.
[ ] 서버 장애 상시 복구 대행이 기본 포함이 아님을 설명했다.
[ ] 유상 지원 범위를 별도로 안내했다.

12. 문서 사용 기준

이 문서는 설치 완료 여부를 짧고 명확하게 확인하기 위한 체크리스트다.
고객과 설치자가 같은 기준으로 납품 상태를 확인할 수 있도록 사용한다.

체크되지 않은 항목은 완료로 간주하지 않는다.
미완료 항목은 납품 전 처리하거나, 고객에게 제외 사유와 후속 조치 범위를 설명한다.
