GridFlow 기준선 동결

목적
- 이번 릴리스 목표는 기능 추가가 아니라 설치형 제품 인수 패키지 완성이다.
- buyer 실사 기준에서 필요한 범위만 고정한다.

이번 릴리스 성공 기준
- 보안 신뢰성
  - 인증, 키 저장, 민감정보 노출 금지 기준이 문서와 실제 구현에서 크게 어긋나지 않아야 한다.
  - auth 저장 방식 이슈는 해소 또는 명시적 HOLD 정리가 필요하다.
- 새 서버 재현 가능성
  - 새 서버에서 설치 문서만으로 필수 서비스와 엔드포인트를 재현할 수 있어야 한다.
  - 숨은 수동 작업이 있으면 문서에 드러나야 한다.
- 핵심 플로우 자동 검증
  - 로그인, 키 반영, 수동주문, Grid, DCA, Rebalancing, monitor의 핵심 경로에 대해 자동 검증 근거가 있어야 한다.
  - 실패 시 silent failure 없이 추적 가능해야 한다.
- buyer-facing 문서/증거 패키지
  - 운영 런북, 설치 체크리스트, 범위 문서, 증거 문서가 buyer에게 바로 설명 가능한 수준으로 정리돼야 한다.
  - 과장 없이 확정/미확정이 구분돼야 한다.

이번 라운드에서 안 할 것
- 새 거래소 추가 안 함
- 새 전략 추가 안 함
- 전면 프론트 리팩터 안 함
- `static/index.html` 대수술 안 함
- 모바일 고도화 안 함
- UI 미세개선 위주 작업 안 함
- 백테스트 확장 안 함
- 예쁜 리팩터링 / 성능 놀이 안 함

딜브레이커
- auth 저장 방식 문제
- 설치 재현 불가
- 백업/복구 실검증 부재
- 핵심 자동 테스트 부재
- buyer 문서/증거 패키지 부재

판정 원칙
- 위 딜브레이커 중 하나라도 해소되지 않으면 이번 릴리스는 확정하지 않는다.
- 문서 간 충돌이 있으면 섣불리 확정하지 않고 추가 확인 필요로 둔다.
- 현재 라운드의 기준선은 “설치형 납품 완성도”이며, 기능 확장 성과로 평가하지 않는다.
